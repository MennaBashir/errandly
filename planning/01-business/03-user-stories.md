# 📋 Core Use Cases (User Stories)

All MVP user stories grouped by epic.

---

## Epic: Authentication & Identity
- **US-A1:** As a user, I want to sign up with phone + OTP so that onboarding is fast.
- **US-A2:** As a Runner, I want to submit ID + selfie for KYC so that I can become payout-eligible.
- **US-A3:** As a user, I want to switch between Master and Runner mode so that I can both post and earn.

## Epic: Task Lifecycle (Master)
- **US-M1:** As a Master, I want to post a task with location, photos, and budget so that Runners can find it.
- **US-M2:** As a Master, I want to **lock money in escrow** so that the Runner trusts they'll be paid and I'm protected from no-shows.
- **US-M3:** As a Master, I want to receive bids and compare Runner ratings so that I pick the best one.
- **US-M4:** As a Master, I want to track my Runner live on a map so that I know progress.
- **US-M5:** As a Master, I want to approve completion only after reviewing proof so that I release money for real work.
- **US-M6:** As a Master, I want to raise a dispute if work is unsatisfactory so that I'm not forced to pay.

## Epic: Task Lifecycle (Runner)
- **US-R1:** As a Runner, I want to see nearby tasks ranked by payout/distance so that I maximize earnings.
- **US-R2:** As a Runner, I want to accept or bid on a task so that I claim work.
- **US-R3:** As a Runner, I want to mark a task complete and upload proof so that escrow can be released.
- **US-R4:** As a Runner, I want to withdraw my wallet balance so that I get real cash.

## Epic: Escrow & Billing
- **US-E1:** As the platform, I must hold funds in escrow from post until approval so that neither party can defraud the other.
- **US-E2:** As the platform, I must deduct exactly 10% on release so that revenue is captured automatically.
- **US-E3:** As the platform, I must refund the Master if a task is cancelled before assignment so funds aren't trapped.

## Epic: Trust, Ratings & Disputes
- **US-T1:** As either party, I want to rate the other 1–5 stars + comment so that reputation is built.
- **US-T2:** As an Admin, I want a dispute resolution console so that I can fairly split/refund escrow.
